## Maintainer responsibilities

Maintainers are responsible for:

- reviewing issues and pull requests
- keeping releases, tags, and PyPI packaging healthy
- enforcing the Code of Conduct
- triaging security reports through the private disclosure process
- protecting Brigade's local-first and review-gated design boundaries

## Project status

Brigade is early-stage open-source infrastructure. The repository is public, MIT licensed, and actively maintained. The current maintainer uses Brigade on real local agent workspaces and treats outside issues, install reports, and reproducible failures as the primary feature queue.

## Open-source program use

Credits, hosted services, or sponsored accounts granted to Brigade should be used only for the public open-source project: documentation, CI, release automation, dependency and security scanning, public community support, and reproducible maintainer workflows. They should not be used for unrelated private consulting, personal projects, or closed commercial work.
